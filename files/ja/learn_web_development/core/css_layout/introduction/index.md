CSS のページレイアウト技術によって、ウェブページに含まれる要素の位置を制御できます。通常のレイアウトフローによる初期配置、隣接する要素、それらの親コンテナー、またはメインビューポート、ウィンドウの位置、といったものを、位置を制御する基準にすることができます。

本モジュールで詳細に説明するページレイアウトの手法には、それぞれ用途、長所、短所があり、単独で使用するようには設計されていません。 ひとつひとつの技術がどのように設計されているかを理解すると、それぞれのタスクに最適なレイアウト方法を判断するための基礎になるでしょう。

## 通常フロー

ウェブページ上の要素は、その動作を変更するような CSS を適用していない限り、**通常フロー**でレイアウトされます。要素の動作は、通常フローにおけるそれらの位置を調整するか、またはフローから完全に削除することで変更できます。通常フローで読みやすい、しっかりとした構造の文書から始めるのが、ウェブページ作成の最善の方法です。これにより、ユーザーが非常に制限されたブラウザーや、ページのコンテンツを読み上げるスクリーンリーダーなどの機器を使用していても、コンテンツが確実に読めるようになります。さらに、標準フローは読みやすい文書を作成するように設計されているため、このやり方で始めれば、レイアウトを変更する際に文書と格闘するのではなく、文書と協力しながら作業を進めることができます。

さまざまなレイアウト方式について深く掘り下げる前に、通常の文書フローについて検討したこれまでのモジュールで学んだことのいくつかを振り返ってみましょう。

## 既定では要素はどのようにレイアウトされるのか

このプロセスは、個々の要素のボックスが、現れるパディング、境界線、マージンがコンテンツに追加されるような方法でレイアウトされることから始まります。これが、私たちが**ボックスモデル**と呼んでいるものです。

既定では、[ブロックレベル要素](/ja/docs/Glossary/Block-level_content)の内容は、それを包含する親要素のインラインで利用できる空間を埋め、そのコンテンツに合わせてブロックの寸法に沿って拡大します。 [インラインレベル要素](/ja/docs/Glossary/Inline-level_content)のサイズは、そのコンテンツのサイズのみです。 {{cssxref("display")}} プロパティの既定の値が `inline` である一部の要素では、 {{cssxref("width")}} または {{cssxref("height")}} を設定できますが（{{HTMLElement("img")}} のように）、 `display` の値は `inline` のままです。

インラインレベル要素の `display` プロパティをこのように制御したい場合は、CSS を使用してブロックレベル要素のように動作するように設定します（例えば、`display: block;` または `display: inline-block;` を使用します。これらは両方の特性を兼ね備えています）。

これは、要素が個別にどのように構造化されるかを説明していますが、それらが互いに作用し合う場合の構造化についてはどうでしょうか？ 通常のレイアウトフロー（レイアウトの紹介記事で言及されている）は、要素がブラウザーのビューポート内に配置される仕組みです。 既定では、ブロックレベル要素はブロックフロー方向にレイアウトされ、これは親の[書字方向](/ja/docs/Web/CSS/writing-mode)（初期設定: horizontal-tb）に基づいています。各要素は、直前の要素の次の新しい行に表示され、指定したマージンで別個の要素として表示されます。例えば英語（または他にも横書きで上から下に書く言語）では、ブロックレベル要素は垂直方向に配置されます。

インライン要素は異なる動作をします。新しい行に現れることはなく、親のブロックレベル要素の幅内に空間がある限り、隣接する（または折り返された）テキストコンテンツとともに、すべて同じ行に配置されます。空間がない場合は、あふれたコンテンツが新しい行に移動されます。

垂直方向に隣接する 2 つの要素のどちらにもマージンが設定されており、そのマージンが接触している場合、 2 つのマージンの中で大きい方のマージンが残り、小さい方のマージンは消えます。これは[**マージンの相殺**](/ja/docs/Web/CSS/CSS_box_model/Mastering_margin_collapsing)と呼ばれています。マージンの相殺は、垂直方向のみに適用されます。

### 通常フローの例

このすべてを説明する単純な例を見ていきましょう。

```html-nolint
<h1>通常文書フロー</h1>

<p>
  これは基本的なブロックレベル要素です。隣接するブロックレベル要素は、この下に新しい行に配置されます。
</p>

<p>
  既定では、親要素の幅を 100% 使用し、子コンテンツの高さと同じ高さになります。全体の幅と高さは、コンテンツの幅と高さ＋パディング＋境界線の幅/高さです。
</p>

<p>
  これらはマージンによって分離されています。マージンが相殺されているため、互いのマージンではなく、マージン 1 つ分のサイズによって分離されています。
</p>

<p>
  この<span>インライン要素</span>や<span>このインライン要素</span>は、同じ行に空間がある場合、隣接するテキストノードとともに同じ行に配置されます。
  インライン要素がオーバーフローすると、<span>可能な場合は（テキストを含むこの要素のように）新しい行に折り返されます</span>。そうでない場合は、この画像のように、新しい行にそのまま続きます。
  <img src="long.jpg" alt="布切れ" />
</p>
```

```css
body {
  width: 500px;
  margin: 0 auto;
}

p {
  background: rgb(255 84 104 / 30%);
  border: 2px solid rgb(255 84 104);
  padding: 10px;
  margin: 10px;
}

span {
  background: white;
  border: 1px solid black;
}
```

HTML は、ソースコードに現れるのと同じ正確な順序で表示され、ブロック要素が互いに積み重ねられていることに注目してください。

ページ上の要素の多くについては、通常のフローで必要なレイアウトが正確に作成されます。しかし、より複雑なレイアウトの場合は、CSS で利用できるツールの一部を使用して、この既定の動作を変更する必要があります。 構造化された HTML 文書から始めることはとても重要です。なぜなら、既定ではレイアウトがどのように構成されているか、それに逆らうのではなく、それを利用しながら作業を進めることができるからです。

## 通常フローの上書き

このモジュールで詳細を説明する、 CSS で通常フローを上書きして、要素をレイアウトする方法を変更できる方式については次のようなものがあります。

- {{cssxref("display")}} プロパティ
  - : `block`、`inline`、`inline-block` などの標準の値は、要素が通常フローでどのようにふるまうかを変更することができ、例えばブロックレベル要素をインラインレベル要素のようにすることができます（詳細については、[ボックスモデル](/ja/docs/Learn_web_development/Core/Styling_basics/Box_model#ブロックボックスとインラインボックス)のレッスンに戻ってください）。
- 浮動ボックス
  - : {{cssxref("float")}} に `left` などの値を適用すると、ブロックレベル要素を要素の片側に沿って折り返すようにすることができ、雑誌のレイアウトで画像の周囲をテキストが取り囲むようにすることができます。
- 位置指定
  - : {{cssxref("position")}} プロパティにより、他のボックス内のボックスの配置を正確に制御できます。 通常フローでは `static` 位置指定が既定ですが、他の値を使用して要素を異なる方法でレイアウトすることもできます。 例えば、ブラウザーのビューポートの上に要素を固定する場合には、 `position: fixed` を使います。
- `display` からアクセスできる具体的なレイアウトシステム
  - : レイアウト方式全体が、 `display` に特定の値を使うことで有効にできる場合もあります。その中でも特に重要なものとして、[CSS グリッド](/ja/docs/Learn_web_development/Core/CSS_layout/Grids)と[フレックスボックス](/ja/docs/Learn_web_development/Core/CSS_layout/Flexbox)どちらも、子要素が親要素内でレイアウトされる方法を変更します。
- レスポンシブデザイン
  - : レスポンシブデザインとは、ウェブページがレンダリングされるさまざまな端末（例えば、デスクトップやモバイル端末）に適応するレイアウトを作成することを指します。 レスポンシブデザインは、それ自身が特定のレイアウトツールを提供するものではありません。最も重要な成分は {{cssxref("@media")}} アットルールであり、画面の幅や解像度などの端末の属性に応じて異なるレイアウトを適用することができます。

### その他のレイアウト手法

他に、それほど一般的ではないレイアウト手法がありますが、このモジュールではそれらには触れません。

- [表レイアウト](/ja/docs/Web/CSS/CSS_table)
  - : HTML の表の一部をスタイル設定するために設計された機能であり、 `display: table` とそれに関連するプロパティを使用して、表以外の要素に使用することもできます。
- [段組みレイアウト](/ja/docs/Web/CSS/CSS_multicol_layout)
  - : 段組みレイアウトのプロパティを使用すると、新聞のようにブロックのコンテンツを段組みにレイアウトできます。
